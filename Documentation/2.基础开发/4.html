    贴图（Sprites）¶

    SoulEngine 借由一个贴图库来创建贴图。
    路径为： Scripts/Libraries/SpritesManager.lua

    创建一个贴图¶

    目前拥有两种方法来创建贴图：

    • sprites.CreateSprite(path, layer) 第一个参数为使用的图片路径 （.png） ，第二个参数为当前的图层大小 （数字） 。
    • sprites.CreateSpriteAtlas(path, x, y, w, h, layer) 本函数的作用是从选中的图片中抠出一部分进行独立的创建。参数分别为：图片路径、区域起始横坐标、区域起始纵坐标、区域宽大小、区域高大小、图层大小。

    示例： 

    -- 这将创建一个以poseur.png为图片的贴图，图层为1.
local s1 = sprites.CreateSprite("poseur.png", 1)

-- 这将创建一个以poseur.png的(10, 10), (60, 60)区域为图片的贴图，图层为2.
local s2 = sprites.CreateSpriteAtlas("poseur.png", 10, 10, 50, 50, 2)

    贴图的属性¶

    有了贴图之后，这个贴图不可能只是单纯的待在一个地方，需要进行操作来达到各种效果。
    下面给出一个贴图拥有的属性：

    • sprite.path 贴图的路径。
    • sprite.parent 贴图的父对象。
    • sprite.realName 贴图的名字，也就是路径去掉 .png 后缀。
    • sprite.isBullet 贴图是否为弹幕类型，若是弹幕类型则可以对玩家造成伤害。
    • sprite.image 贴图的图片素材。
    • sprite.layer 贴图的图层。
    • sprite.dust 贴图化尘的变量。
    • sprite.visible 贴图是否可以被看见。
    • sprite.isactive 贴图是否还在正常运作。
    • sprite.alpha 贴图的透明度。(0 ~ 1)
    • sprite.color 贴图的颜色。 {r, g, b} (0 ~ 1)
    • sprite.width 贴图的图片宽度。
    • sprite.height 贴图的图片高度。
    • sprite.xpivot 贴图的水平锚点（默认0.5，居中位置）。
    • sprite.ypivot 贴图的竖直锚点（默认0.5，居中位置）。
    • sprite.rotation 贴图的旋转角度。
    • sprite.x 贴图的x坐标。
    • sprite.y 贴图的y坐标。
    • sprite.relx 贴图的相对x坐标，用于parent。
    • sprite.rely 贴图的相对y坐标，用于parent。
    • sprite.velocity 贴图的移动速度。
    • sprite.speed 贴图的绝对速度。此变量一般不用于外部。
    • sprite.xshear 贴图的水平斜切。
    • sprite.yshear 贴图的竖直斜切。
    • sprite.xscale 贴图的水平缩放倍数。
    • sprite.yscale 贴图的竖直缩放倍数。
    • sprite.animation 贴图的动画。

    再接下来给出贴图的方法：

    • MoveTo(x, y) 让贴图移动到 (x, y) 的位置。
    • Move(x, y) 让贴图以 (x, y) 的速度进行移动。
    • SetSpeed(x, y) 设置贴图的移动速度（多用于蓝魂的板子）。
    • Scale(x, y) 设置贴图的缩放。
    • Pivot(x, y) 设置贴图的锚点。
    • Shear(x, y) 设置贴图的斜切。
    • Set(path) 改变一个贴图的图片为路径位置的图片。
    • GetPosition() 获得贴图的位置。
    • SetAnimation(frames, interval, mode) 设置贴图的动画，间隔的帧数，动画模式。
    • SetStencils(stencils) 设置贴图的遮罩。
    • SetShaders(shaders) 设置贴图使用的shaders。
    • SetParent(parent) 设置贴图的父对象（一般需要父对象有x, y）。
    • Dust(sound) 贴图化尘，设置是否播放音效。
    • Destroy() 销毁一个贴图。
    • Remove() 同 Destroy() ，销毁一个贴图。

    贴图方法的详细介绍¶

    上面已经说过了一个贴图的方法都有哪些，便于迅速查询，接下来进行实际应用以及解释：

    MoveTo(x, y) -> nil¶

    将一个贴图移动到 (x, y) 的位置，什么也不返回。 

    sprite:MoveTo(320, 120)

    Move(x, y) -> nil¶

    令一个贴图以 (x, y) 的速度进行移动，什么也不返回。 

    sprite:Move(2, 0)

    SetSpeed(x, y) -> nil¶

    将一个贴图的速度设置为 (x, y) ，如果想要停下来这个贴图则需要重新使用这个函数来进行设置。 

    sprite:SetSpeed(3, 1)
-- 自从上面设置以后，贴图总会按着这一个速度移动下去。

-- 下面这段代码会让它停下来。
sprite:SetSpeed(0, 0)

    Scale(x, y) -> nil¶

    将一个贴图以水平竖直方向分别为 (x, y) 的倍数进行缩放。 

    -- 贴图会在水平方向上变为10倍，竖直方向上变为2倍。
sprite:Scale(10, 2)

    Pivot(x, y) -> nil¶

    设置一个贴图的锚点位置为 (x, y) 。
    默认值是 (0.5, 0.5) ，如果是 (0, 0) 那么在左上角，如果是 (1, 1) 那么在右下角。 

    sprite:Pivot(0, 0.5)

    Shear(x, y) -> nil¶

    设置贴图的水平与竖直斜切。 

    sprite:Shear(0.2, 0) -- 水平斜切0.2

    Set(path) -> nil¶

    将贴图的图片改为 Resources/Sprites/ 下的指定图片，并设置为最近邻过滤。 

    sprite:Set("poseur.png")

    GetPosition() -> x, y¶

    返回贴图的当前位置坐标。 

    local x, y = sprite:GetPosition()

    SetAnimation(frames, interval, mode) -> nil¶

    为贴图设置动画帧表、帧间隔与播放模式（默认 "loop"）。

    所有的播放模式和描述如下：

    • loop：循环模式，播放完之后自动从头再来一遍。
    • oneshot：不是 oneshot 游戏，指的是播放完之后自动停留在最后一帧，之后可以自由设置这个贴图。
    • oneshot-empty：播放完所有序列之后自动销毁，适用于Flowey从地下钻出或钻回地下。 
      sprite:SetAnimation({"a.png", "b.png"}, 6, "loop")

    SetStencils(stencils) -> nil¶

    设置贴图使用的遮罩源列表；若列表为空则禁用遮罩。 

    sprite:SetStencils({mask1, mask2})

    SetShaders(shaders) -> nil¶

    设置贴图使用的 shader 列表；若为空则禁用。会把当前颜色（包含 alpha）通过 sColor 发送到每个 shader。 

    sprite:SetShaders({shader1, shader2})

    SetParent(parent) -> nil¶

    设置贴图的父对象（父对象应包含 x, y 等），传入 nil 可取消父子关系并保持位置为独立。 

    sprite:SetParent(parentObject)
sprite:SetParent(nil) -- 取消父对象

    Dust(sound) -> nil¶

    触发贴图化尘效果：创建 dust shader、画布并准备迭代图像。若传入 true 会播放音效 "snd_dust.wav"。 

    sprite:Dust(true)

    Destroy() -> nil¶

    销毁贴图：标记不活动、释放与 dust 相关的 shader/画布资源，并从 sprites.images 与 layers.objects 中移除该贴图。 

    sprite:Destroy()

    Remove() -> nil¶

    同 Destroy()，销毁贴图的别名。 

    sprite:Remove()

    贴图转弹幕¶

    正如上面所说，将 sprite.isBullet 变量设置为 true ，可以将一个贴图转化成为弹幕，此时这个贴图将具有攻击性。

    local bul = sprites.CreateSprite("bullet.png", 0)
bul.isBullet = true

    弹幕的碰撞¶

    默认碰撞¶

    所有弹幕的碰撞在默认情况下都是一个矩形，每个矩形的属性包括：

    • 位置： (x, y)
    • 大小： (w, h)
    • 角度： r

    对于默认情况，判定玩家碰撞是否生效，采用的是矩形和点的位置关系来进行计算。

    完美像素碰撞¶

    此功能于 SoulEngine 2.5.0 首次登场，可以判定任何形状的弹幕用于碰撞。

    给每个转化成弹幕的贴图使用 SetPPCollision(true) 即可让此贴图使用完美像素碰撞。

    local bul = sprites.CreateSprite("bullet.png", 0)
bul.isBullet = true

-- 这个弹幕将使用精确碰撞
bul:SetPPCollision(true)

    精确碰撞的危险之处：如果一个特别大的贴图，或者一个拥有很多复杂透明区域的贴图，开启了完美碰撞之后，会造成游戏严重卡顿，有时甚至会导致游戏立刻闪退。

    当然，不必担心，特别大在这里指的是 1000*1000 及以上的尺寸。在实际制作游戏的过程中，我们几乎不会用到这个尺度的图片，请放心大胆地对其他弹幕使用精确碰撞。
