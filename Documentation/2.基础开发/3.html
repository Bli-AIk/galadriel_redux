    场景（Scene）¶

    对于只有 一个全局环境的 LOVE2D来说，多场景独立让场景成为了 SoulEngine 的最大特色。

    2.5.1 速递：扩充了场景内置变量的说明。

    关于场景的基本信息¶

    在 SoulEngine 中，每个场景都是一个独立的文件，其结构基本与 main.lua 的结构相似。
    其拥有和 main.lua 一样的函数，只是所有者不同。

    下面是一个默认场景的代码文件内容： 

    -- This is a template for creating a new scene in the game.
-- You can use this as a starting point for your own scenes.
local SCENE = {
    PRIORITY = true
}

-- This is a fake scene for testing purposes.
function SCENE.load()
    -- Load any resources needed for this scene here.
    -- For example, you might load images, sounds, etc.
end

-- This function is called to update the scene.
function SCENE.update(dt)
    -- Update any game logic for this scene here.
    -- For example, you might update animations, handle input, etc.
end

-- This function is called to draw the scene.
-- It is called after the main game loop has finished updating.
function SCENE.draw()
    -- Draw the scene here.
    -- For example, you might draw images, text, etc.
end

-- This function is called when the scene is switched away from.
function SCENE.clear()
    -- Clear any resources used by this scene here.
    -- For example, you might unload images, sounds, etc.
    layers.clear()
end

-- Don't touch this(just one line).
return SCENE
    抛开其他的注释不谈，我们可以只看看带颜色的部分。
    这些带颜色的部分就是一个场景最基本的函数。

    接下来我们挨个解释：

    • local SCENE = {}：这是此场景的“容器”。
    • PRIORITY = true：此场景是否优先于贴图库进行绘制操作。
    • SCENE.load()：场景被激活的时候调用，但是请注意，不要在这里面初始化后续会需要使用的变量，否则必须使用全局变量，在场景内请尽可能使用局部变量。
    • SCENE.update(dt)：用于进行一个场景的持续更新，可以在这里放置自增计时器，dt的作用是赋值为delta_time参与计算。
    • SCENE.draw()：用于进行一个场景的绘制操作，你可以在这里绘制图片，文本等内容，也可以用于DEBUG测试绘制。
    • SCENE.clear()：这是 本场景的新增功能 ，在退出场景时，有些东西总会被删掉的，所以其内部含有一个自带的layers.clear函数，用于在离开场景时的删除元素功能。
    • return SCENE：返回这个场景容器，用于在main.lua中进行使用。

    如果要创建一个属于自己的场景，请直接复制这个基本场景代码，然后加载，你会进入一个完全黑色的房间，这就是本引擎的空房间。

    场景内置变量¶

    在场景中，有一些隐藏的变量没有被展示，除了上文提到的 PRIORITY 之外，还有一些其他的变量：

    • PRIORITY：设置之后会让场景优先级别高于贴图。
    • SAVESHADERS：默认关闭，也就是说如果切换场景，那么所有的屏幕Shaders都会被清除，打开后可在切换场景之间保留Shaders。
    • PERSISTENT：默认关闭，是否保留场景当前状态，如果开启那么下次回到场景还是原先的样子（目前这个变量并不稳定，需要静待之后的调整）。

    main.lua如何使用一个场景¶

    既然引擎里有这么多场景，那么你是否很疑惑，为什么切换到了这个场景，那这个场景就会自己更新？
    要回答这个问题其实很简单，答案就在于：LOVE2D引擎只有一个 全局环境 ，这会让我们处理每个场景容易得多。

    首先，让我们把目光聚焦于 main.lua 的顶端： 

    scenes = require("Scripts.Libraries.SceneManager")
    这部分代码就是调用一个 场景管理器 的东西，我们之后再说。
    这会让我们获取到 scenes.switchTo() 这个方法，用于场景切换。

    然后，把目光聚焦到 love.load 函数中： 

    local success, err = pcall(function()
    scenes.switchTo("Overworld/scene_ow_new") -- Start with the logo scene.
    reset_room = "scene_logo" -- Which room will the player switch to after pressed f2.
end)

if (not success) then
    error(err)
end
    说简单点，就是这里面只有一个 scenes.switchTo 函数有用，这会加载路径为 Scripts/Scenes/Overworld/scene_ow_new 的场景文件。

    实际上，在使用了 switchTo 功能之后，有一个变量就被存储到了 scenes.current 中，也就是 当前场景 。

    所以，我们继续往下看，就可以看到 love.update 函数中有 scenes.current.update(dt) 的调用： 

    if (scenes.current) then
    if (scenes.current.update) then
        scenes.current.update(dt)
    end
end
    有时候，你可能需要在这里修改一下其他库和场景库的更新顺序，可能会有不同的结果。

    接下来就是 love.draw 函数内部的代码，这段代码将这些东西放置在了 CANVAS 上，而后我们会进行绘制 CANVAS 的操作来让所有元素正常显现： 

    love.graphics.push()

    -- 应用摄像机的缩放。
    love.graphics.scale(scale, scale)
    gui.draw()

    _CAMERA_:apply()
    love.graphics.setColor(1, 1, 1)

    -- 如果现在存在scenes.current这个当前场景（也就是说如果现在有正在运行的场景）
    if (scenes.current) then
        if (not scenes.current.PRIORITY) then
            if scenes.current and scenes.current.draw then
                -- 优先绘制场景内部
                scenes.current.draw()
            end

            -- 再绘制全局贴图
            layers.sort()
        else

            -- 优先绘制全局贴图
            layers.sort()
            if scenes.current and scenes.current.draw then
                -- 再绘制场景内部
                scenes.current.draw()
            end
        end
    end

    _CAMERA_:reset()

love.graphics.pop()
    通过代码可以发现，场景可以拥有一个 PRIORITY 变量，这个变量控制着优先度，用来决定到底是先绘制场景、还是先绘制全局。

    例： 

    local SCENE = {
    PRIORITY = false
}

-- ......
    这会让 贴图库（SpritesManager.lua） 创建的贴图与 文本库（TyperManager.lua） 创建的文本优先于场景元素进行绘制。

    后续是每个场景的接口和 main.lua 接口，无需进行处理。

    默认场景¶

    SoulEngine 默认有一些已经搭建好的场景，这里进行简便的解释说明（注：路径均以 Scripts/Scenes/ 为起点）：

    • scene_logo.lua 标题页面，也是进入游戏后的第一个场景。
    • scene_battle_new.lua 战斗场景。 new 表明是经历过重大变革后投入使用的。所有的战斗场景均可以通过复制此场景创建。
    • scene_battle_ow.lua ow专属的战斗场景，拥有提升LOVE的能力，不过现阶段已废弃。
    • scene_battle_spider.lua 蜘蛛战斗场景。在Overworld中对蜘蛛选择丢弃即可进入。
    • scene_end.lua 结束场景，可以用来存放DEMO END页面，目前被用于测试贴图化尘效果。
    • scene_gameover.lua 游戏结束场景，当玩家的HP为0时进入。
    • scene_gameover_b.lua Overworld专用结束场景，会回到上一个存档的位置。
    • scene_template.lua 场景模板，创建新场景时请复制一份。

    下面是 Overworld 专区：

    • scene_ow_new.lua 默认Overworld场景，用于测试以后Overworld中所需要的所有东西。
    • scene_ow_ruins_1.lua 第二个Overworld场景。

    场景的注意事项¶

    1. 一定要注意：初次进入场景的时候重置一些变量。
    2. 注意 clear 函数内的内容，一定要使用 package.loaded 操作清除在这里所require过的所有库。
    3. 尽可能的使用 local 字段的局部变量。

    场景详解¶

    在这里，你可以找到如何使用一些重要的默认场景的方法，以及理解整个场景的运作方式。

    战斗场景（scene_battle_new.lua）¶

    经历革新之后，战斗场景变的十分简洁。
    所以需要额外讲解的东西并不多。
    接下来介绍一些重要函数：

    • HandleActions(enemy_name, action) 用于控制 ACT 选项，传入参数为 玩家选择的敌人的名字 + 玩家的选项 。
    • HandleItems(itemID) 当玩家选择一个物品的时候，会在这里调用，传入参数为 物品的名字 。请注意，就算物品有标签，也要带着写进去。
    • HandleSpare() 当玩家选择仁慈时调用。在这里可以做出仁慈跳转到 ATTACKING 页面然后直接攻击敌人的效果。
    • HandleFlee() 当玩家选择逃跑且 成功 的时候调用，这里可以做出 Disbelief Papyrus（全然不信） 中玩家逃跑然后被拉回的效果。
    • DefenseEnding() 在玩家离开回合的 一瞬间 调用，多用于写切换回合的逻辑。
    • OnHit(Bullet) 当玩家碰到弹幕的时候调用，关于一个弹幕如何让玩家受伤的逻辑也在这里面。

    主世界场景（scene_ow_new.lua）¶

    与战斗场景相似，经历过重大革新后，整个场景变得十分简单。
    这里只介绍一些比较常用的函数或者功能。

    • _CAMERA_:setBounds(minX, minY, maxX, maxY) 设置一个摄像机的边界，保证摄像机不会超出范围。
    • ow.InitMusic("mus_house1.ogg") 设置一个overworld的场景的默认BGM为 mus_house1.ogg 。若切换到的场景和此场景有同样的BGM，那么音乐不会从头播放或中断，而是继续播放。
    • ow.Init("Maps/ruins_0.lua", "scene_ow_new") 依据一个lua tiled文件来加载一个overworld地图。第二个参数的作用是本文件的位置。
    • ow.SetBattleScene() 设置一个ow场景遭遇怪物后要去的战斗场景。
    • ow.DEBUG 是否开启Overworld的开发者模式。开发者模式将会在Overworld主世界专题中进行介绍。
    • ow.ENCOUNTER 是否开启本地图遇怪系统，若关闭则本场景不会遭遇怪物。
    • ow.Update(dt) Overworld的更新逻辑。
    • ow.getInteractResult("save", 1) 当玩家和地图内一个物体接触时调用，需在其中写按下确认键来制作一个 触碰 + 确认 的交互流程。
    • ow.SaveInteract() 保存初始化设置。
    • ow.ChangeScene() 切换到某个场景的某个地标点。
    • ow.ChestInteract() 和什么箱子交互。
    • ow.dialogNew() Overworld专属的创建对话的函数。
    • ow.Draw() Overworld的绘制功能。

    以上这些内容还只是Overworld的冰山一角，在Overworld专题中会重新详细论述这些内容。

    根据上面的所有内容，你可以开始选择一个场景进行创作了。
