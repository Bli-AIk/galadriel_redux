    使用地图¶

    2.5.1 速递：新增了 ow.SetBattleScene 的第二位参数，用于填写flag。

    导出了地图之后，最重要的就是使用它。下面我们来新建一个Overworld场景然后使用它吧。

    初始化场景¶

    首先，我们需要给Overworld场景准备一个绝佳的位置，这里我推荐复制我下面代码块所写的所有内容，直接复制，一个字符都不要漏掉。

    打开之后，复制粘贴： 

    -- This is a template for creating a new scene in the game.
-- You can use this as a starting point for your own scenes.
local SCENE = {}

--_CAMERA_:setBounds(280 - 320, 300 - 240, 280 + 320, 300 + 240)
local ow = require("Scripts.Libraries.Overworld.InitWorld")
ow.InitMusic("mus_gameover.ogg")
ow.Init("Maps/ruins_1.lua", "scene_ow_ruins_1")
ow.DEBUG = true
ow.ENCOUNTER = true
ow.InitEncounter(FLAG.ruins_killed, 80, 40, 3)
ow.SetBattleScene("Battle/scene_battle_ow", "ruins_killed")

-- This is a fake scene for testing purposes.
function SCENE.load()
    -- Load any resources needed for this scene here.
    -- For example, you might load images, sounds, etc.
end

-- This function is called to update the scene.
function SCENE.update(dt)
    -- Update any game logic for this scene here.
    -- For example, you might update animations, handle input, etc.
    ow.Update(dt)

    if (ow.NEXTSTATE ~= nil) then
        ow.CSTATE = ow.NEXTSTATE
        ow.NEXTSTATE = nil
    end
end

-- This function is called to draw the scene.
-- It is called after the main game loop has finished updating.
function SCENE.draw()
    -- Draw the scene here.
    -- For example, you might draw images, text, etc.
    ow.Draw()
end

-- This function is called when the scene is switched away from.
function SCENE.clear()
    -- Clear any resources used by this scene here.
    -- For example, you might unload images, sounds, etc.
    package.loaded["Scripts.Libraries.Overworld.InitWorld"] = nil

    layers.clear()
    ow.Destroy()
end

-- Don't touch this(just one line).
return SCENE

    让我们来好好看看这里发生了什么事情。

    首先自动舍弃掉从 load 函数开始之后的所有内容，让我们看看初始化的部分： 

    --_CAMERA_:setBounds(280 - 320, 300 - 240, 280 + 320, 300 + 240)
local ow = require("Scripts.Libraries.Overworld.InitWorld")
ow.InitMusic("mus_gameover.ogg")
ow.Init("Maps/ruins_1.lua", "scene_ow_ruins_1")
ow.DEBUG = true
ow.ENCOUNTER = true
ow.InitEncounter(FLAG.ruins_killed, 80, 40, 3)

    • ow = require：调用 Overworld 初始化库，没了这个Overworld是无法建立起来的。
    • ow.InitMusic("mus_gameover.ogg")：设置这个房间的音乐为 mus_gameover.ogg 。
    • ow.Init("Maps/ruins_1.lua", "scene_ow_ruins_1")：加载 Maps/ruins_1.lua 的地图，整个房间地址为 scene_ow_ruins_1 。
    • ow.DEBUG = true：目前是开启DEBUG的状态。
    • ow.ENCOUNTER = true：打开后，玩家在地图里游荡会遭遇怪物，进入战斗。
    • ow.InitEncounter(FLAG.ruins_killed, 80, 40, 3)：玩家的杀怪路线目前由ruins_killed记录数目，以80开始，随机浮动40f，总共遭遇三次怪物。

    很明显，想要使用自己的地图，就只需要改ow.Init函数中的内容即可。

    玩家交互物体¶

    使用 ow.getInteractResult(type, id) 来获取一个玩家和给定物体的交互结果。

    • type：物体的类型是什么，比如 trigger, chest, save 等，均为图层名称去掉复数形式。
    • id：物体的id是什么，这个id是在Tiled中被设置过的，用来区分唯一的物体。

    比如 ow.getInteractResult("trigger", 1) 就会判定玩家和trigger类型id为1的物体是否碰撞。

    例子： 

    if (ow.getInteractResult("trigger", 1)) then
    if (keyboard.GetState("confirm") == 1) then
        print("You triggered smth.")
    end
end
    上面这个案例所描述的内容是，如果玩家碰到了id为1的trigger，并且按下了确认键，那么将会触发一些东西。

    遭遇怪物¶

    设置遭遇信息¶

    使用函数 ow.InitEncounter(flag, start, range, amount) 来初始化整个刷怪系统设置。

    • flag：由谁来掌管这个刷怪数目（一般都由FLAG里面的一个变量进行控制）。
    • start：最低刷怪时间是多少，也就是说当玩家必须至少走过start的时间才可以开始遇到怪物。
    • range：浮动范围区间，玩家会在最大start + range的时间遇到怪物。
    • amount：一共有多少个怪物要杀，会自动根据flag的数目调整刷怪时长，当怪物越少，时间倍数越大，但最大不会超过8倍，防止游玩周期过长。

    如果你对原版的刷怪时长公式感兴趣，可以看看下面的说明：

    首先是遭遇密度系数ρ
ρ = min(8, amount / (amount − killed))

当剩余怪数量(amount-killed)越来越少时，ρ会越来越大，但是最大不会超过8.

目标时间阈值
t = ρ*(base + U(0, range))

U(0, range)表示在[0, range]范围内均匀分布的随机数
每次结束战斗后重新计算这个值

    设置遭遇场景¶

    使用 ow.SetBattleScene(scene, flag) 来设置一个玩家遭遇怪物后会去的战斗场景。

    ow.SetBattleScene("Battle/scene_battle_ow", "ruins_killed")
    上述代码表述为当玩家遭遇怪物时，会进入 Battle/scene_battle_ow 这个场景。并且使用 FLAG.ruins_killed 作为监视变量，来检测玩家是否杀掉敌人。

    特定交互¶

    创建对话¶

    使用 ow.dialogNew(texts, position) 来生成一个对话框，并打出texts中的内容。

    position参数是这个对话框在上方还是下方，如果不填写那么会自动寻找合适的位置，如果填写则强制使用填写的位置。

    ow.dialogNew({"* Nothing useful here."})

    创建箱子交互¶

    使用 ow.ChestInteract(chest) 来让玩家和一个箱子进行交互。

    参数 chest 是第几个箱子的表，举例： 

    ow.ChestInteract("chest1")
    上述代码会让玩家打开CHESTS.chest1的箱子进行互动。

    创建存档点交互¶

    使用 ow.SaveInteract(texts, room, position, direction) 来让玩家和一个存档点交互。

    • texts：点击存档点之后输入的话。
    • room：存档地点的名称，比如 最后的长廊 。该描述最终会显示在存档页面。
    • posiition：玩家通过存档进入游戏位于什么位置。
    • direction：玩家通过存档进入游戏面朝哪里。

    ow.SaveInteract(
    {
        "[colorHEX:99ffff]* BAKABAKA",
    },
    "Idk where", {121, -455}, "down"
)
    当玩家点击存档点后，会先输入 BAKABAKA ，随后房间地点为 Idk where ，玩家的位置被保存在了 {121, -455} ，玩家的面朝方向被保存为了 down 。

    切换房间¶

    使用 ow.ChangeScene(scene, mark) 来让玩家切换到一个房间的地标点处。

    ow.ChangeScene("Overworld/scene_ow_ruins_1", 1)
