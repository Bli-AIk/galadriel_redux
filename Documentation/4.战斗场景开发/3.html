    回合的开发¶

    作为传说之下的同人，一定少不了回合的。
    在这里我一般称作回合为 wave ，你也可以叫它 turn ，这两个单词意思在这里差不多。

    和战斗场景的联系¶

    所有的回合位置均位于 Scripts/Waves/ 文件夹中，一个战斗场景如果想要获取一个回合，那么它就会从这里面寻找对应的路径。

    对于一般情况下，我们会在 DefenseEnding 函数中设置回合推进。 

    local nextwaves = {"wave_test1", "wave_test2", "wave_test3", "wave_test4"}
b.battle.nextwave = "wave_test1"

local waveProgress = 1

local function DefenseEnding()
    waveProgress = waveProgress + 1
    if (waveProgress > #nextwaves) then waveProgress = 1 end

b.battle.nextwave = nextwaves[waveProgress]
end

    上面所说的内容是，在 nextwaves 中，设置好所有的回合序列，随后优先通过 b.battle.nextwave 设置第一个回合是谁。

    waveProgress 变量负责检测回合推进的情况，调成第一个回合（当你想要制作首尾相连的回合的时候可以限制这个变量的值）。一般情况下，我们只需要 b.battle.nextwave = nextwaves[waveProgress] 来一个回合一个回合慢慢推进即可。

    内容速览¶

    让我们快速看看回合里面几条有用的东西说了什么：

    local Arena = battle.mainarena
local Player = battle.Player

local wave = {
    ENDED = false,
    objects = {}
}

local function EndWave()
    wave.ENDED = true
    arenas.clear()
    for i = #wave.objects, 1, -1 do
        local obj = wave.objects[i]
        obj:Destroy()
        table.remove(wave.objects, i)
    end
end

Arena:Resize(155, 130)
Player.sprite:MoveTo(320, 320)

local mask = masks.New("rectangle", 320, 320, 155, 130, 0, 1)

function wave.update(dt)
    mask:Follow(Arena.black)

    for i = #wave.objects, 1, -1 do
        local obj = wave.objects[i]
        if (obj.logic) then
            obj:logic(dt)
        end
    end
end

function wave.draw()
end

return wave
    • Arena = battle.mainarena：这是战斗的主战斗框，由于新时期同人研究出了加减框这个神奇的东西，分清谁是主战斗框尤为重要，而这个值就是 battle.mainarena 负责。但是如果一直操纵 battle.mainarena 很难看，整个代码会很长，所以现在，只需要说一个 Arena ，代码就知道你指的是主战斗框。
    • Player = battle.Player：和上面Arena的原因一致。代指玩家的主灵魂。
    • EndWave()：最重要的函数，可以结束当前回合，回到主线。
    • Arena:Resize(155, 130)：当进入回合后，把战斗框的尺寸改成(155, 130)。
    • Player.sprite:MoveTo(320, 320)：进入回合后，将玩家的灵魂放在战斗框的正中间。
    • mask = masks.New：详情请见遮罩部分的 内容。

    战斗框¶

    所有战斗框的结构都是黑叠白结构，也就是是先绘制的白色矩形，后在上面盖了一个黑色矩形。

    总体来说呢，战斗框还有不少方法的。

    战斗框的方法¶

    • Arena:Resize(w, h)：会让战斗框设置一个 目标尺寸 ，然后战斗框会缩放到那里。
    • Arena:MoveTo(x, y)：会让战斗框设置一个 目标位置 ，移动到那里。
    • Arena:MoveToAndResize(x, y, w, h)：移动并缩放（把上面两个流程整合起来了）。
    • Arena:UpSide(value)：保持战斗框原位置不变，单独向上拓展value个像素。
    • Arena:DownSide, LeftSide, RightSide：同理。
    • RotateTo(rotation)：设置目标角度，旋转过去。
    • Arena:Destroy()：销毁战斗框（不要给主战斗框用这个！）。

    瞬间改变战斗框的属性¶

    也许你在使用的过程中已经发现了，上面这些东西都不能立刻改变战斗框的属性，必须要先看一遍动画才行。

    不过幸运的是，我们有直接改变属性的方法。

    由于战斗框可以设置目标属性和当前属性（比如设置目标尺寸和当前尺寸），我们可以直接利用这个手写目标属性。

    -- 只能设置目标尺寸，还得看动画过渡
Arena:Resize(155, 130)

    -- 现在既然设置了目标尺寸，直接设置当前的尺寸也没有任何的问题
Arena:Resize(155, 130)
Arena.width = 155
Arena.height = 130

    骨头¶

    是的我知道你们很喜欢骨头，接下来让我们创建一个骨头吧。

    2D骨头的创建¶

    由于回合中没有能够创建骨头的地方，我们需要先 require 库。 

    local bones = require("Scripts.Libraries.Attacks.Bones")
    在这之后，我们可以开始创建骨头了。

    使用 bones:New2D(whose, length) 来创建一根骨头：

    • whose：谁的骨头，只有两个参数可以选择，Sans, Papyrus。
    • length：这根骨头多长，需要注意这个是加上了骨头头和尾的长度，一般减去12就是你想要的长度。
    -- 这根骨头就可以恰好填满宽度为130的战斗框
local bone = bones:New2D("Sans", 130 - 12)

    2D骨头的方法¶

    诚然，有些骨头也是只会显示在战斗框里的，它们不会出框，这是遮罩起了作用，那么骨头也可以设置遮罩等一系列内容：

    • bone:SetStencils(masks)：设置骨头的遮罩。
    • bone:Length(length)：改变骨头的长度，只能大于等于0。
    • bone:SetColor(color)：设置一整根骨头的颜色（不是攻击类型，攻击类型需要后面这个函数来设置）。
    • bone:SetMode(mode)：设置骨头的攻击类型，是橙色、蓝色、普通等等......
    • bone:ToUp(arena)：假设你创建了一根骨头，你想让它跟踪某个战斗框的顶端，那么就需要用 ToUp(Arena) （这里Arena仍旧是主战斗框）来让骨头一端紧贴上边。
    • ToDown, ToLeft, ToRight：同理。
    • bone:Destroy()：销毁自身。

    3D骨立方的创建¶

    是的，上面的部分只是2D的骨头，也就是平时用的骨头，接下来是炫技用的3D骨头。

    首先，我们明确了解一下，3D骨头并不是直接创建的3D骨头，而是先创建立方体，后对两个点两两连线做出的3D骨立方。

    所以现在我们先创建一个立方体：

    local cube = bones:New3D({
    {320 - 100, 240 - 100, 50}, {320 + 100, 240 - 100, 50},
    {320 - 100, 240 + 100, 50}, {320 + 100, 240 + 100, 50},
    {320 - 100, 240 - 100, 250}, {320 + 100, 240 - 100, 250},
    {320 - 100, 240 + 100, 250}, {320 + 100, 240 + 100, 250}
})

    我选取的中心点就是(320, 240, 150)，当然，中心点不需要手动输入，代码会自动计算这几个点的平均值来作为中心点。

    3D骨立方连线¶

    由于我们创建了八个点，这八个点也被存储进了这个 cube 变量中，接下来，我们使用 cube:Link(whose, p1, p2) 来连接这八个点，一共12条棱。

    cube:Link("Sans", 1, 2)
cube:Link("Sans", 1, 3)
cube:Link("Sans", 2, 4)
cube:Link("Sans", 3, 4)
cube:Link("Sans", 1 + 4, 2 + 4)
cube:Link("Sans", 1 + 4, 3 + 4)
cube:Link("Sans", 2 + 4, 4 + 4)
cube:Link("Sans", 3 + 4, 4 + 4)
cube:Link("Sans", 1, 5)
cube:Link("Sans", 2, 6)
cube:Link("Sans", 3, 7)
cube:Link("Sans", 4, 8)

    所以接下来，你会看见骨立方成功连线了。

    但是，可能有些时候，这些骨头太长太长，导致容错太低了，这时候我们就需要一个百分比取舍，来规定这些连线的长度是多少倍。

    cube.percent = 0.4

    这样，所有的骨头长度都是之前的0.4倍了。

    3D骨立方的方法¶

    当然，3D骨头也是拥有方法的：

    • Link(whose, p1, p2)：不再介绍
    • Rotate(axis, angle)：绕哪个轴进行旋转，axis一共有三个参数可选 x, y, z，angle则是转多少度。
    • MoveTo(x, y)：移动骨立方。
    • MoveToRelative(x, y)：移动骨立方的中心点。
    • SetStencils(masks)：设置骨立方的遮罩。
    • Destroy()：销毁自身。

    骨墙¶

    使用 bones:Wall(arena, whose, warntime, staytime, length, direction, rotation, interval, animation) 来创建一个骨墙。

    • arena：哪个战斗框。
    • whose：谁的骨头。
    • warntime：警告时间。
    • staytime：骨墙伸出后停驻时间。
    • length：伸出多长。
    • direction：有四个参数 up, down, left, right来确定从哪个边伸出。
    • rotation：暂时无法设置。
    • interval：暂时无法设置。
    • animation：骨墙动画，一个骨墙的伸出动画、时间，退回动画、时间分别是多少。

    示例： 

    local wall = bones:Wall(battle.mainarena, "Papyrus", 40, 30, 50, "down", 10, 13, {
    In = "SineOut", Out = "BackIn",
    It = 15         , Ot  = 60
})
wall:SetStencils({mask})

    目前骨墙仅有一个设置遮罩的方法可用，无其余方法。

    GasterBlaster¶

    使用函数 blasters:New(begin, final, angles, waittime, firingtime, sprite_prefix, sounds_prefix) 创建一个GB炮。

    • begin：开始位置
    • final：最终位置
    • angles：具体格式为 {0, 90}，就是从0度到90度。
    • waittime：等待多少时间开火（建议不要小于40）。
    • firingtime：开火多长时间。
    • sprite_prefix：贴图路径指向。
    • sounds_prefix：音效路径指向。
    local gb1 = blasters:New(
    {0, 0}, {320, 240},
    {180, 0},

    40, 10,

    nil,
    nil
)

    回调函数¶

    在GB开火的时候，有一个被藏起来的函数，会在那一瞬间触发一次。

    让我们延续gb1，来这样写代码：

    gb1.TimeCall = function()
    print("开火了")
end

    你会发现，终端里确实出现开火了这三个字，所以说我们可以掌控开火的这一瞬间内容。

    恰好我们又有一个函数，叫做Beam，这是GB独有的方法，于是我们还可以这么写：

    gb1.TimeCall = function()
    gb1:Beam(30)
end
    这会在开火的时候，加装一个以30°角射出的激光。

    我们还可以把原本的激光取消掉，变成哑炮。

    gb1.default_fire = false
