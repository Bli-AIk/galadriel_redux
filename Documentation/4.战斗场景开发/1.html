    认识战斗场景¶

    在 SoulEngine 中，玩家遭遇一个战斗，会进入到一个特定的场景之中，所有的战斗逻辑，都写在了这个场景里面。

    战斗场景的位置¶

    所有的战斗场景，均被放置在了 Scenes/Battle/ 文件夹中，使用的是 scene_battle 这个场景制作战斗。

    也许你也发现了还有一个场景，叫做 scene_battle_origin ，这是老版本的战斗场景，内部逻辑量极大，很容易导致 upvalues 超出阈值的报错，出于这个原因， scene_battle 是 origin 版本的精简版，它只提供了几个接口，便可以完成整场战斗的制作。

    但是，这并不代表着 scene_battle_origin 就没有任何作用了，由于这个场景过于底层，当你想要全面修改整个战斗逻辑中的元素，你就可以直接使用这个场景进行修改。

    接下来，我们看看 scene_battle 里面有什么。

    战斗场景的内容¶

    下面展示的是 scene_battle 的默认代码： 

    -- This is a template for creating a new scene in the game.
-- You can use this as a starting point for your own scenes.
local SCENE = {}
local b = require("Scripts.Libraries.Battle.BattleInit")
battle = b.Init("Scripts.Libraries.Game.Encounter")
atkp = require("Scripts.Libraries.Battle.Patterns.template")
b.SetAtkPattern(atkp)

local function HandleActions(enemy_name, action)
    local battle = b.battle
    local temp_tab = {}

    -- Deep copy
    for k, v in pairs(battle.Enemies) do
        temp_tab[k] = v
    end

    if not battle or not battle.Enemies then return end
    if (enemy_name == temp_tab[1].name) then
        local enemy = battle.Enemies[1]
        if (action == enemy.actions[1]) then
            b.BattleDialogue(localize.Act11)
        elseif (action == enemy.actions[2]) then
            b.BattleDialogue(localize.Act12)
        end
    elseif (enemy_name == temp_tab[2].name) then
        local enemy = battle.Enemies[2]
        if (action == enemy.actions[1]) then
            b.BattleDialogue(localize.Act21)
        elseif (action == enemy.actions[2]) then
            b.BattleDialogue(localize.Act22)
        end
    end
end

local function HandleItems(itemID)
    local battle = b.battle
    if not battle then return end
    local inventory = battle.Inventory
    local randomText = {
        "* No."
    }

    b.BattleDialogue({
        "* You found an item...[wait:30]\n  [colorRGB:255, 255, 0]" .. itemID .. "!",
        randomText[love.math.random(1, #randomText)]
    })
end

local function HandleSpare()
    b.battle.STATE = "ACTIONSELECT"
end

local nextwaves = {"wave_test1", "wave_test2", "wave_test3", "wave_test4"}
b.battle.nextwave = "wave_test1"
local waveProgress = 1
local function DefenseEnding()
    waveProgress = waveProgress + 1
    if (waveProgress > #nextwaves) then waveProgress = 1 end
    b.battle.nextwave = nextwaves[waveProgress]
end

local function OnHit(Bullet)
    local battle = b.battle
    if not battle then return end
    local mode = Bullet['HurtMode']
    if (mode == "normal" or type(mode) == "nil") then
        battle.Player.Hurt(1, 60, true)
        b.AddKR(5)
    elseif (mode == "cyan" or mode == "blue") then
        if (keyboard.GetState("arrows") > 0) then
            battle.Player.Hurt(1, 0, true)
            b.AddKR(1)
        end
    elseif (mode == "orange") then
        if (keyboard.GetState("arrows") <= 0) then
            battle.Player.Hurt(1, 0, true)
            b.AddKR(1)
        end
    elseif (mode == "green") then
        battle.Player.Heal(1)
        Bullet:Destroy()
    end
end

b.HandleActions = HandleActions
b.HandleItems   = HandleItems
b.HandleSpare   = HandleSpare
b.DefenseEnding = DefenseEnding
b.OnHit         = OnHit





-- This is a fake scene for testing purposes.
function SCENE.load()
    -- Load any resources needed for this scene here.
    -- For example, you might load images, sounds, etc.
end

-- This function is called to update the scene.
function SCENE.update(dt)
    -- Update any game logic for this scene here.
    -- For example, you might update animations, handle input, etc.
    b.Update(dt)
    if (b.GetSelectedEnemy() == 1 and b.GetState() == "ATTACKING") then
        -- print("Selected Poseur")
    end
end

-- This function is called to draw the scene.
-- It is called after the main game loop has finished updating.
function SCENE.draw()
    -- Draw the scene here.
    -- For example, you might draw images, text, etc.
    b.Draw()
end

-- This function is called when the scene is switched away from.
function SCENE.clear()
    -- Clear any resources used by this scene here.
    -- For example, you might unload images, sounds, etc.
    b.Clear()
    package.loaded["Scripts.Libraries.Battle.BattleInit"] = nil
end

-- Don't touch this(just one line).
return SCENE

    战斗系统初始化¶

    local b = require("Scripts.Libraries.Battle.BattleInit")
battle = b.Init("Scripts.Libraries.Game.Encounter")
atkp = require("Scripts.Libraries.Battle.Patterns.template")
b.SetAtkPattern(atkp)
    第一行：调用战斗场景框架。
    第二行：这场战斗要面对的敌人是哪个文件来的，这里的内容也包括玩家初始的血量，名字等内容。（制作的时候需要修改）
    第三行：玩家的攻击方式是什么。（如果没有手动修改那么默认就是木棍和真刀的攻击动画）。
    第四行：给战斗系统声明一下我要用第三行的这个攻击样式了。

    回调函数¶

    这些回调函数在玩家执行特定操作的时候会被触发，注意下面所描述的触发时机来制作出自己想要的效果：

    • HandleActions(enemy_name, action)：当选中 enemy_name 名字的敌人而且选中的行动为 action 时，执行这里面的内容。
    • HandleItems(itemID)：当选择名字为 itemID 的物品时，执行里面的内容。
    • HandleSpare()：当玩家选择Spare按钮的时候，执行里面的内容。
    • DefenseEnding()：当玩家从敌人回合中回到自己回合的时候执行里面的内容。
    • OnHit(Bullet)：当玩家和一个具有攻击性的贴图 isBullet = true 进行碰撞时，执行里面的内容。

    下面这些内容没有直接出现在 scene_battle 里，但还是可以被修改的：

    • HandleFlee()：当玩家逃跑成功的一瞬间触发。
    • FleeUpdate()：玩家逃跑成功之后持续执行此函数。

    内置函数¶

    这些函数有的可以检测玩家攻击的是什么怪物、有的可以在战斗框直接打出特定内容（旁白），下面是这些所有内置函数的列举：

    • b.BattleDialogue(texts, targetState)：让旁白打字机打出 texts 中的内容，随后在打完所有内容之后，让状态自动跳转到 targetState 。比如你想打完对话后，还回到玩家选择四个按钮的地方，这里的 targetState 就填写 ACTIONSELECT ；如果想要直接进入战斗，就填写 DEFENDING 。
    • b.AddKR(kramount)：给玩家添加kr。
    • b.SetAtkPattern(pattern)：设置战斗攻击动画。
    • b.SetWinText(func)：手动设置战斗胜利之后的文本。
    • b.GetUIButtons()：获取底部四个按钮（这四个按钮会被放到一个表里）。
    • b.GetSelectedEnemy()：获取玩家现在选择的是第几个敌人。
    • b.GetSelectedEnemyName()：获取玩家现在选择的敌人是什么名字。
    • b.GetState()：获取现在是什么状态。
    • b.WinBattleCall()：胜利的一瞬间执行这个函数（里面有上述的SetWinText设置胜利文本）。
    • b.ToggleKR()：开关KR的函数，若在有KR的情况下关闭了KR，那么所有剩余的KR会转化成血量。
    • b.EnteringStateInherited()：EnteringState函数的继承，在单独场景中使用请进行设置。
    • b.EnemyJustDied()：当在场有任意一个敌人被杀死时，触发此函数。

    战斗场景变量¶

    也许你好奇Sans的开头杀是怎么制作的，应该如何迅速进入第一个场景，又或者如何找到敌人的位置。

    敌人的设置¶

    所有的敌人文件都位于 Scripts/Libraries/Game/文件夹中，玩家的一些基础变量比如名字、血量都在这里。

    战斗状态（STATE）¶

    STATE是比较复杂的一个系统，拥有多个结果来定位目前的UI状态，下面是所有STATE的列举：

    一层状态菜单，可通过单次按键达到的地方：

    • ACTIONSELECT：玩家回合，选择四个按钮的状态。
    • FIGHTMENU：战斗菜单，当玩家点击FIGHT按钮的时候STATE会变成FIGHTMENU。
    • ACTMENU：同理，点击的是ACT按钮。
    • ITEMMENU：同理。
    • MERCYMENU：同理。
    • DEFENDING：敌人回合的状态。

    二层状态菜单，可能需要从上一个状态进入当前状态：

    • ATTACKING：玩家开始校准、根据时机按下攻击条的状态。需要从 FIGHTMENU 进入。
    • ACTSELECTING：玩家在行动中选择了敌人，准备选择对敌人的行动。需要从 ACTMENU 进入。
    • DIALOGUERESULT：进入了战斗框的旁白状态（通常是使用了物品或者选择了行动）。

    STATE全分支详解（首尾循环状态部分加星号）：

    无开头杀： 

    ACTIONSELECT* -> FIGHTMENU -> ATTACKING -> DEFENDING -> ACTIONSELECT*

    有开头杀： 

    DEFENDING* -> ACTIONSELECT -> FIGHTMENU -> ATTACKING -> DEFENDING*

    选择行动： 

    ACTIONSELECT* -> ACTMENU -> ACTSELECTING -> DIALOGUERESULT -> ACTIONSELECT*

    选择物品： 

    ACTIONSELECT* -> ITEMMENU -> DIALOGUERESULT -> ACTIONSELECT*

    选择仁慈： 

    ACTIONSELECT* -> MERCYMENU -> ACTIONSELECT*(/DEFENDING -> ACTIONSELECT*)

    选择逃跑： 

    ACTIONSELECT* -> MERCYMENU -> FLEEING(/DEFENDING) -> ACTIONSELECT*
