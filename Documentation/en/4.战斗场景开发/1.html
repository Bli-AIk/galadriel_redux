    Understanding Battle Scenes¶

    In SoulEngine, when the player encounters a battle, they enter a dedicated scene. All battle logic is written inside that scene.

    Battle Scene Location¶

    All battle scenes are placed in the Scenes/Battle/ folder and use the scene_battle scene to build battles.

    You may also notice another scene named scene_battle_origin. This is the old version of the battle scene. Its internal logic is extremely large and can easily trigger an upvalues threshold error. For this reason, scene_battle is a simplified version of scene_battle_origin. It only provides a few interfaces, but it is enough to create a full battle.

    That does not mean scene_battle_origin is useless. Because it is much lower level, it is still the best place to modify the entire battle flow when you want to change the system from the ground up.

    Now let's see what is inside scene_battle.

    Battle Scene Contents¶

    The default code from scene_battle is shown below: 

    -- This is a template for creating a new scene in the game.
-- You can use this as a starting point for your own scenes.
local SCENE = {}
local b = require("Scripts.Libraries.Battle.BattleInit")
battle = b.Init("Scripts.Libraries.Game.Encounter")
atkp = require("Scripts.Libraries.Battle.Patterns.template")
b.SetAtkPattern(atkp)

local function HandleActions(enemy_name, action)
    local battle = b.battle
    local temp_tab = {}

    -- Deep copy
    for k, v in pairs(battle.Enemies) do
        temp_tab[k] = v
    end

    if not battle or not battle.Enemies then return end
    if (enemy_name == temp_tab[1].name) then
        local enemy = battle.Enemies[1]
        if (action == enemy.actions[1]) then
            b.BattleDialogue(localize.Act11)
        elseif (action == enemy.actions[2]) then
            b.BattleDialogue(localize.Act12)
        end
    elseif (enemy_name == temp_tab[2].name) then
        local enemy = battle.Enemies[2]
        if (action == enemy.actions[1]) then
            b.BattleDialogue(localize.Act21)
        elseif (action == enemy.actions[2]) then
            b.BattleDialogue(localize.Act22)
        end
    end
end

local function HandleItems(itemID)
    local battle = b.battle
    if not battle then return end
    local inventory = battle.Inventory
    local randomText = {
        "* No."
    }

    b.BattleDialogue({
        "* You found an item...[wait:30]
  [colorRGB:255, 255, 0]" .. itemID .. "!",
        randomText[love.math.random(1, #randomText)]
    })
end

local function HandleSpare()
    b.battle.STATE = "ACTIONSELECT"
end

local nextwaves = {"wave_test1", "wave_test2", "wave_test3", "wave_test4"}
b.battle.nextwave = "wave_test1"
local waveProgress = 1
local function DefenseEnding()
    waveProgress = waveProgress + 1
    if (waveProgress > #nextwaves) then waveProgress = 1 end
    b.battle.nextwave = nextwaves[waveProgress]
end

local function OnHit(Bullet)
    local battle = b.battle
    if not battle then return end
    local mode = Bullet['HurtMode']
    if (mode == "normal" or type(mode) == "nil") then
        battle.Player.Hurt(1, 60, true)
        b.AddKR(5)
    elseif (mode == "cyan" or mode == "blue") then
        if (keyboard.GetState("arrows") > 0) then
            battle.Player.Hurt(1, 0, true)
            b.AddKR(1)
        end
    elseif (mode == "orange") then
        if (keyboard.GetState("arrows") <= 0) then
            battle.Player.Hurt(1, 0, true)
            b.AddKR(1)
        end
    elseif (mode == "green") then
        battle.Player.Heal(1)
        Bullet:Destroy()
    end
end

b.HandleActions = HandleActions
b.HandleItems   = HandleItems
b.HandleSpare   = HandleSpare
b.DefenseEnding = DefenseEnding
b.OnHit         = OnHit




-- This is a fake scene for testing purposes.
function SCENE.load()
    -- Load any resources needed for this scene here.
    -- For example, you might load images, sounds, etc.
end

-- This function is called to update the scene.
function SCENE.update(dt)
    -- Update any game logic for this scene here.
    -- For example, you might update animations, handle input, etc.
    b.Update(dt)
    if (b.GetSelectedEnemy() == 1 and b.GetState() == "ATTACKING") then
        -- print("Selected Poseur")
    end
end

-- This function is called to draw the scene.
-- It is called after the main game loop has finished updating.
function SCENE.draw()
    -- Draw the scene here.
    -- For example, you might draw images, text, etc.
    b.Draw()
end

-- This function is called when the scene is switched away from.
function SCENE.clear()
    -- Clear any resources used by this scene here.
    -- For example, you might unload images, sounds, etc.
    b.Clear()
    package.loaded["Scripts.Libraries.Battle.BattleInit"] = nil
end

-- Don't touch this(just one line).
return SCENE

    Battle System Initialization¶

    local b = require("Scripts.Libraries.Battle.BattleInit")
battle = b.Init("Scripts.Libraries.Game.Encounter")
atkp = require("Scripts.Libraries.Battle.Patterns.template")
b.SetAtkPattern(atkp)
    Line 1: call the battle scene framework.
    Line 2: specify which enemy file this battle should face. This file also includes the player's initial HP, name, and other values. (Change this when you build the encounter.)
    Line 3: define the player's attack type. If you do not change this manually, the default is the wooden stick / knife attack animation.
    Line 4: tell the battle system to use the attack pattern declared on line 3.

    Callback Functions¶

    These callback functions are triggered when the player performs specific actions. Pay attention to the trigger timing below to create the effect you want:

    • HandleActions(enemy_name, action): triggered when the enemy with name enemy_name is selected and the selected action is action.
    • HandleItems(itemID): triggered when the item with ID itemID is selected.
    • HandleSpare(): triggered when the player chooses the Spare button.
    • DefenseEnding(): triggered when the game returns from the enemy turn to the player's turn.
    • OnHit(Bullet): triggered when the player collides with a projectile-style sprite where isBullet = true.

    The following callbacks do not appear directly inside scene_battle, but they can still be modified:

    • HandleFlee(): triggered the instant the player successfully flees.
    • FleeUpdate(): continues to run after the player has fled successfully.

    Built-in Functions¶

    Some of these functions can detect which enemy the player is attacking; some can print text directly in the battle box. Here is the full list:

    • b.BattleDialogue(texts, targetState): makes the narration type out texts, then automatically switches to targetState after the text finishes. For example, if you want to return to the four-button menu after the text, set targetState to ACTIONSELECT; if you want to go straight into battle, set it to DEFENDING.
    • b.AddKR(kramount): add KR to the player.
    • b.SetAtkPattern(pattern): set the battle attack pattern.
    • b.SetWinText(func): manually set the victory text after winning.
    • b.GetUIButtons(): retrieve the four buttons at the bottom of the screen (they are stored in a table).
    • b.GetSelectedEnemy(): get the index of the currently selected enemy.
    • b.GetSelectedEnemyName(): get the name of the currently selected enemy.
    • b.GetState(): get the current state.
    • b.WinBattleCall(): triggered at the moment of victory (this function uses the text set by SetWinText).
    • b.ToggleKR(): toggle KR on or off. If KR is turned off while the player has KR, the remaining KR is converted into HP.
    • b.EnteringStateInherited(): inherit the EnteringState function. Set this when using the scene separately.
    • b.EnemyJustDied(): This function is triggered when any enemy on the field is killed.

    Battle Scene Variables¶

    You may be curious how Sans's opening attack is made, how to quickly enter the first scene, or how to find enemy positions.

    Enemy Settings¶

    All enemy files are located in Scripts/Libraries/Game/. The player's base variables such as name and HP are also stored there.

    Battle State (STATE)¶

    STATE is a relatively complex system. It has multiple values that describe the current UI state. Below are all STATE values:

    A first-level state menu that can be reached with a single input:

    • ACTIONSELECT: player turn; choose one of four buttons.
    • FIGHTMENU: battle menu when the player clicks FIGHT.
    • ACTMENU: battle menu when the player clicks ACT.
    • ITEMMENU: battle menu when the player clicks ITEM.
    • MERCYMENU: battle menu when the player clicks MERCY.
    • DEFENDING: enemy turn.

    A second-level state menu that is usually entered from the previous state:

    • ATTACKING: the player is aiming and timing the attack bar. This is entered from FIGHTMENU.
    • ACTSELECTING: the player has chosen an action and is selecting the enemy target. This is entered from ACTMENU.
    • DIALOGUERESULT: the battle dialog box is active (usually after using an item or selecting an action).

    All STATE branches explained (states with loops at the beginning/end are marked with an asterisk):

    No opening attack: 

    ACTIONSELECT* -> FIGHTMENU -> ATTACKING -> DEFENDING -> ACTIONSELECT*

    With opening attack: 

    DEFENDING* -> ACTIONSELECT -> FIGHTMENU -> ATTACKING -> DEFENDING*

    Choose an action: 

    ACTIONSELECT* -> ACTMENU -> ACTSELECTING -> DIALOGUERESULT -> ACTIONSELECT*

    Choose items: 

    ACTIONSELECT* -> ITEMMENU -> DIALOGUERESULT -> ACTIONSELECT*

    Choose mercy: 

    ACTIONSELECT* -> MERCYMENU -> ACTIONSELECT*(/DEFENDING -> ACTIONSELECT*)
