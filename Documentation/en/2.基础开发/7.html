    Audio¶

    Used for playing music and sound effects.

    Playing music¶

    To handle large audio files smoothly, LOVE2D allows streaming playback, loading part of the audio while playing it.

    This avoids sudden freezes caused by loading large files all at once.

    Use audio.PlayMusic(music, volume, looping) -> Source, table to play audio from Resources/Music/.

    Example: 

    -- Play mus_gameover.ogg at 50% volume, looping
audio.PlayMusic("mus_gameover.ogg", 0.5, true)

    Playing sound effects¶

    Sound effects are usually short audio clips. LOVE2D can load these statically.

    Use audio.PlaySound(sound, volume, looping) -> Source, table to play audio from Resources/Sounds/.

    Example: 

    -- Play snd_ding.wav at 80% volume
audio.PlaySound("snd_ding.wav", 0.8)

    Advanced usage¶

    Note that the two functions above return not only a table, but also a Source. The Source is the audio source, while the table is the manager.

    Usually using a single variable returns only the source. The manager is used for advanced techniques or low-level modification.

    -- Source only
local mus = audio.PlayMusic("a.ogg", 1, true)

-- Source and manager
local mus, ins = audio.PlayMusic("a.ogg", 1, true)

    Volume fade¶

    Use VolumeTransition(begin, target, duration) to change volume over time.

    Example: 

    local mus, ins = audio.PlayMusic("mus_gameover.ogg", 0, true)
ins:VolumeTransition(0, 1, 1.4)

    Pitch fade¶

    Use PitchTransition(begin, target, duration) to change pitch over time.

    The structure is the same as VolumeTransition, so we will not repeat it.

    Loop points¶

    Use SetLoopPoints(startTime, endTime) to set loop points on an audio source.

    When playback reaches endTime, it jumps back to startTime, creating a loop section. To disable looping, set ins.loopStart to false.

    Audio effects¶

    Use SetEffect(effect_name, params) to apply an effect to an audio source.

    Note: some devices may not support audio effects, so results may not change.

    Audio effects¶

    The following effects are available. First, ignore the effect_name parameter string; any string is allowed. You only need to use the expected lowercase effect name.

    local myreverb = {
    type = "reverb"
}
ins:SetEffect("myr", myreverb)

    Reverb¶

    Simulates sound reflections in different environments such as halls or caves.

    • (number) gain: overall gain (volume), default 0.316.
    • (number) gainhf: high-frequency gain decay factor, default 0.89.
    • (number) room: room size base gain, default 0.5.
    • (number) roomhf: high-frequency factor for room size, default 0.5.
    • (number) decay: reverb decay time in seconds, default 1.49.
    • (number) decayhf: high-frequency decay relative to low frequency, default 0.83.
    • (number) reflectionsgain: early reflection gain, default 0.05.
    • (number) reflectionsdelay: early reflection delay in seconds, default 0.007.
    • (table) reflectionspan: early reflection direction vector, default {0.0, 0.0, 0.0}.
    • (number) lategain: late reverb gain, default 1.26.
    • (number) latedelay: late reverb delay in seconds, default 0.011.
    • (table) latepan: late reverb direction vector, default {0.0, 0.0, 0.0}.
    • (number) diffusion: diffusion, default 1.0.
    • (number) density: density, default 1.0.
    • (number) hfreference: high frequency reference in Hz, default 5000.0.
    • (number) lfreference: low frequency reference in Hz, default 250.0.
    • (number) roomrolloff: room size distance attenuation factor, default 0.0.
    • (number) airborneabsorbtiongainhf: high-frequency air absorption gain, default 0.994.

    Chorus¶

    Produces a chorus effect by mixing delayed copies of the sound.

    • (number) rate: modulation frequency in Hz, default 1.1.
    • (number) depth: modulation depth, default 0.1.
    • (number) feedback: feedback amount, default 0.25.
    • (number) delay: base delay time in seconds, default 0.016.
    • (table) pan: stereo direction vector, default {0.0, 0.0, 0.0}.

    Distortion¶

    Produces overdrive and distortion by clipping the waveform.

    • (number) gain: input gain before distortion, default 0.05.
    • (number) edge: distortion edge hardness, default 0.2.
    • (number) posteqcenterfrequency: post-distortion equalizer center frequency in Hz, default 2400.0.
    • (number) posteqbandwidth: post-distortion equalizer bandwidth, default 2400.0.
    • (number) prelowpasscutoff: pre-distortion lowpass cutoff frequency in Hz, default 8000.0.

    Echo¶

    Simulates repeated sound reflections.

    • (number) delay: echo delay time in seconds, default 0.1.
    • (number) lrdelay: left/right channel delay difference in seconds, default 0.1.
    • (number) damping: high-frequency attenuation per echo, default 0.5.
    • (number) feedback: feedback amount controlling echo duration, default 0.5.
    • (number) spread: stereo spread, default -1.0.

    Equalizer¶

    Changes tone by adjusting gain for different frequency bands.

    • (number) lowgain: low frequency gain, default 1.0.
    • (number) lowcutoff: low frequency cutoff in Hz, default 200.0.
    • (number) mid1gain: low-mid gain, default 1.0.
    • (number) mid1center: low-mid center frequency in Hz, default 500.0.
    • (number) mid1width: low-mid bandwidth, default 1.0.
    • (number) mid2gain: high-mid gain, default 1.0.
    • (number) mid2center: high-mid center frequency in Hz, default 3000.0.
    • (number) mid2width: high-mid bandwidth, default 1.0.
    • (number) highgain: high frequency gain, default 1.0.
    • (number) highcutoff: high frequency cutoff in Hz, default 6000.0.

    Flanger¶

    Produces a swirling effect by mixing a slightly delayed signal with periodic delay variation.

    • (number) rate: modulation frequency in Hz, default 0.1.
    • (number) depth: modulation depth, default 1.0.
    • (number) feedback: feedback amount, default -0.5.
    • (number) delay: base delay time in seconds, default 0.002.

    Ring Modulator¶

    Multiplies the input signal with an internal oscillator to create metallic or robotic tones.

    • (number) frequency: oscillator frequency in Hz, default 440.0.
    • (number) highpasscutoff: highpass filter cutoff in Hz, default 800.0.
    • (number) waveform: oscillator waveform type, 0 for sine, 1 for square, default 0.

    Pitch Shifter¶

    Changes pitch without altering playback speed.

    • (number) coarsetune: coarse pitch in semitones, default 0.
    • (number) finetune: fine pitch adjustment in cents, default 0.

    Compressor¶

    Reduces dynamic range by lowering loud signals and raising quiet signals.

    • (number) onoff: switch. 1 to enable, 0 to disable, default 0.

    MusicGroup¶

    Music groups manage multiple music segments for combined playback. They are useful for: - intro + loop + outro structures - dynamic BGM switching - story or battle music control

    Create¶

    audio.CreateMusicGroup(name, config, volume)

    config: 

    start = "intro.ogg"
loops = {"loop1.ogg", "loop2.ogg"}
outro = "outro.ogg"

    Example: 

    audio.CreateMusicGroup("battle", {
    start = "battle_intro.ogg",
    loops = {"battle_loop1.ogg", "battle_loop2.ogg"},
    outro = "battle_outro.ogg"
}, 1)
