    Sprites¶

    SoulEngine uses a sprite manager to create sprites.

    Path: Scripts/Libraries/SpritesManager.lua

    Create a sprite¶

    There are currently two ways to create a sprite:

    • sprites.CreateSprite(path, layer): first parameter is the image path (.png), second parameter is the layer.
    • sprites.CreateSpriteAtlas(path, x, y, w, h, layer): creates a sprite from part of an image. Parameters are image path, region x, region y, region width, region height, and layer.

    Example: 

    -- This creates a sprite using poseur.png on layer 1.
local s1 = sprites.CreateSprite("poseur.png", 1)

-- This creates a sprite from the region (10, 10) size (50, 50) of poseur.png on layer 2.
local s2 = sprites.CreateSpriteAtlas("poseur.png", 10, 10, 50, 50, 2)

    Sprite properties¶

    After creating a sprite, it can do more than just sit still. Here are its properties:

    • sprite.path: the sprite's path.
    • sprite.parent: the sprite's parent object.
    • sprite.realName: the sprite's name, derived from the path without the .png extension.
    • sprite.isBullet: whether the sprite is a bullet type; bullets can damage the player.
    • sprite.image: the sprite's image resource.
    • sprite.layer: the sprite's layer.
    • sprite.dust: dust effect variable.
    • sprite.visible: whether the sprite is visible.
    • sprite.isactive: whether the sprite is still active.
    • sprite.alpha: sprite opacity (0 to 1).
    • sprite.color: sprite color {r, g, b} (0 to 1).
    • sprite.width: sprite image width.
    • sprite.height: sprite image height.
    • sprite.xpivot: horizontal anchor point (default 0.5 = centered).
    • sprite.ypivot: vertical anchor point (default 0.5 = centered).
    • sprite.rotation: sprite rotation angle.
    • sprite.x: sprite x position.
    • sprite.y: sprite y position.
    • sprite.relx: sprite relative x for parent binding.
    • sprite.rely: sprite relative y for parent binding.
    • sprite.velocity: sprite velocity.
    • sprite.speed: absolute sprite speed. This field is generally not used externally.
    • sprite.xshear: horizontal shear.
    • sprite.yshear: vertical shear.
    • sprite.xscale: horizontal scale.
    • sprite.yscale: vertical scale.
    • sprite.animation: sprite animation.

    Next are sprite methods:

    • MoveTo(x, y): move the sprite to position (x, y).
    • Move(x, y): move the sprite with velocity (x, y).
    • SetSpeed(x, y): set the sprite's movement speed (often used by blue soul platforms).
    • Scale(x, y): scale the sprite.
    • Pivot(x, y): set the sprite's pivot.
    • Shear(x, y): set the sprite's shear.
    • Set(path): change the sprite's image to the one at the given path.
    • GetPosition(): get the sprite's position.
    • SetAnimation(frames, interval, mode): set animation frames, interval, and play mode.
    • SetStencils(stencils): set sprite stencils.
    • SetShaders(shaders): set sprite shaders.
    • SetParent(parent): set the sprite's parent object (the parent should have x and y).
    • Dust(sound): apply a dust effect and optionally play sound.
    • Destroy(): destroy the sprite.
    • Remove(): alias for Destroy().

    Sprite method details¶

    Above is a quick reference for sprite methods. The following explains how to use them.

    MoveTo(x, y) -> nil¶

    Move the sprite to position (x, y). 

    sprite:MoveTo(320, 120)

    Move(x, y) -> nil¶

    Move the sprite with velocity (x, y). 

    sprite:Move(2, 0)

    SetSpeed(x, y) -> nil¶

    Set the sprite's speed to (x, y). To stop the sprite, call this function again with zeros. 

    sprite:SetSpeed(3, 1)
-- After this, the sprite moves at the given speed.

-- Stop the sprite.
sprite:SetSpeed(0, 0)

    Scale(x, y) -> nil¶

    Scale the sprite by (x, y) horizontally and vertically. 

    -- The sprite becomes 10 times wider and 2 times taller.
sprite:Scale(10, 2)

    Pivot(x, y) -> nil¶

    Set the sprite pivot point to (x, y).
    The default is (0.5, 0.5). If you set (0, 0), the pivot is the top-left corner; if (1, 1), it is the bottom-right corner. 

    sprite:Pivot(0, 0.5)

    Shear(x, y) -> nil¶

    Set the sprite shear values. 

    sprite:Shear(0.2, 0) -- horizontal shear of 0.2

    Set(path) -> nil¶

    Change the sprite's image to the specified file under Resources/Sprites/, using nearest-neighbor filtering. 

    sprite:Set("poseur.png")

    GetPosition() -> x, y¶

    Return the sprite's current position. 

    local x, y = sprite:GetPosition()

    SetAnimation(frames, interval, mode) -> nil¶

    Set animation frames, frame interval, and playback mode (default is "loop").

    Modes:

    • loop: loops back to the start after completion.
    • oneshot: plays once and stops on the last frame.
    • oneshot-empty: destroys the sprite after animation finishes, useful for Flowey's emergence or disappearance. 
      sprite:SetAnimation({"a.png", "b.png"}, 6, "loop")

    SetStencils(stencils) -> nil¶

    Set the sprite's stencil source list. If the list is empty, stencils are disabled. 

    sprite:SetStencils({mask1, mask2})

    SetShaders(shaders) -> nil¶

    Set the sprite's shader list. If empty, shaders are disabled. The current color (including alpha) is forwarded via sColor to each shader. 

    sprite:SetShaders({shader1, shader2})

    SetParent(parent) -> nil¶

    Set the sprite's parent object (the parent should have x, y, etc.). Passing nil removes the parent while preserving position. 

    sprite:SetParent(parentObject)
sprite:SetParent(nil) -- remove the parent

    Dust(sound) -> nil¶

    Trigger the sprite dust effect: create a dust shader and canvas, then prepare the dust rendering. If true, it also plays sound snd_dust.wav. 

    sprite:Dust(true)

    Destroy() -> nil¶

    Destroy the sprite: mark it inactive, release dust-related shader/canvas resources, and remove it from sprites.images and layers.objects. 

    sprite:Destroy()

    Remove() -> nil¶

    Alias for Destroy(). 

    sprite:Remove()
