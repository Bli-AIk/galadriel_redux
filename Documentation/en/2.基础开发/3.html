    Scene¶

    For LOVE2D, which only has one global environment, independent multiple scenes are SoulEngine's core feature.

    2.5.1 update: expanded explanation of built-in scene variables.

    Basic scene information¶

    In SoulEngine, each scene is an independent file with a structure similar to main.lua. It has the same type of functions, but a different owner.

    Here is a default scene file template: 

    -- This is a template for creating a new scene in the game.
-- You can use this as a starting point for your own scenes.
local SCENE = {
    PRIORITY = true
}

-- This is a fake scene for testing purposes.
function SCENE.load()
    -- Load any resources needed for this scene here.
    -- For example, you might load images, sounds, etc.
end

-- This function is called to update the scene.
function SCENE.update(dt)
    -- Update any game logic for this scene here.
    -- For example, you might update animations, handle input, etc.
end

-- This function is called to draw the scene.
-- It is called after the main game loop has finished updating.
function SCENE.draw()
    -- Draw the scene here.
    -- For example, you might draw images, text, etc.
end

-- This function is called when the scene is switched away from.
function SCENE.clear()
    -- Clear any resources used by this scene here.
    -- For example, you might unload images, sounds, etc.
    layers.clear()
end

-- Don't touch this (just one line).
return SCENE

    Putting aside the comments, the colored parts show the scene's basic functions.

    Now we explain them one by one:

    • local SCENE = {}: this is the scene container.
    • PRIORITY = true: whether this scene has priority over sprite drawing.
    • SCENE.load(): called when the scene is activated. Avoid initializing variables here that will be needed later unless they are global; use local variables inside the scene when possible.
    • SCENE.update(dt): the scene's continuous update function. You can use it for timers and logic; dt is the delta time value.
    • SCENE.draw(): the scene's draw function. Use it to render images, text, or debugging visuals.
    • SCENE.clear(): this is a newly added scene function. When exiting a scene, some elements must be removed, and it contains a built-in layers.clear call to clean them up.
    • return SCENE: return this scene container for use in main.lua.

    If you want to create your own scene, copy this basic template and load it. You will enter a fully black room, which is the engine's empty scene.

    Built-in scene variables¶

    There are some hidden scene variables beyond PRIORITY:

    • PRIORITY: when set, the scene is drawn before sprites.
    • SAVESHADERS: default is off. If enabled, screen shaders are preserved during scene switches.
    • PERSISTENT: default is off. If enabled, the scene preserves its state so it appears the same when revisited (this feature is currently unstable and may be revised later).

    How main.lua uses a scene¶

    If the engine has many scenes, you may wonder why the current scene updates automatically. The answer is simple: LOVE2D has only one global environment, which makes scene management easier.

    First, look at the top of main.lua: 

    scenes = require("Scripts.Libraries.SceneManager")
    This code loads the scene manager. It gives us scenes.switchTo(), which is used to change scenes.

    Then look at love.load: 

    local success, err = pcall(function()
    scenes.switchTo("Overworld/scene_ow_new") -- Start with the logo scene.
    reset_room = "scene_logo" -- Which room will the player switch to after pressed f2.
end)

if (not success) then
    error(err)
end
    In short, only scenes.switchTo matters here. It loads the scene file at Scripts/Scenes/Overworld/scene_ow_new.

    After switching scenes, the current scene is stored in scenes.current. That's the current scene.

    Now look at love.update: 

    if (scenes.current) then
    if (scenes.current.update) then
        scenes.current.update(dt)
    end
end
    Sometimes you may need to adjust the update order between other libraries and the scene manager. That can change behavior.

    Next is love.draw, which renders everything to CANVAS before displaying it: 

    love.graphics.push()

    -- Apply camera scaling.
    love.graphics.scale(scale, scale)
    gui.draw()

    _CAMERA_:apply()
    love.graphics.setColor(1, 1, 1)

    if (scenes.current) then
        if (not scenes.current.PRIORITY) then
            if scenes.current and scenes.current.draw then
                scenes.current.draw()
            end

            layers.sort()
        else
            layers.sort()
            if scenes.current and scenes.current.draw then
                scenes.current.draw()
            end
        end
    end

    _CAMERA_:reset()

love.graphics.pop()
    This code shows that a scene can set PRIORITY to control whether the scene is drawn before or after global sprites.

    Example: 

    local SCENE = {
    PRIORITY = false
}
    This makes sprites from SpritesManager.lua and text from TyperManager.lua draw before the scene.

    Later sections describe scene interfaces and main.lua interactions. No further action is needed.

    Default scenes¶

    SoulEngine includes some built-in scenes. Below is a brief explanation (paths are relative to Scripts/Scenes/):

    • scene_logo.lua: title screen, the first scene after startup.
    • scene_battle_new.lua: battle scene. new indicates it was introduced after major improvements.
    • scene_battle_ow.lua: Overworld-specific battle scene with enhanced LOVE features, now deprecated.
    • scene_battle_spider.lua: spider battle scene. In Overworld, you can enter it by discarding the spider.
    • scene_end.lua: ending scene, used for demo end screens and testing dust effects.
    • scene_gameover.lua: game over scene, entered when player HP reaches 0.
    • scene_gameover_b.lua: Overworld-specific game over scene, returns to the previous save location.
    • scene_template.lua: scene template. Copy it when creating a new scene.

    Overworld scenes:

    • scene_ow_new.lua: default Overworld scene for testing all Overworld features.
    • scene_ow_ruins_1.lua: second Overworld scene.

    Scene notes¶

    1. Be careful to reset certain variables the first time a scene is entered.
    2. In the clear function, use package.loaded to unload all required libraries for that scene.
    3. Use local variables whenever possible.

    Scene details¶

    Here you can learn how to use important default scenes and understand how the scene system works.

    Battle scene (scene_battle_new.lua)¶

    After major improvements, the battle scene has become much simpler. So there is not much extra explanation needed. Here are some important functions:

    • HandleActions(enemy_name, action): handles ACT menu options. Pass the enemy name and the player's selected option.
    • HandleItems(itemID): called when the player selects an item. Pass the item name, including any tags.
    • HandleSpare(): called when the player selects MERCY. Use it to transition to ATTACKING and then attack the enemy directly.
    • HandleFlee(): called when the player successfully flees. Use it to create effects like Disbelief Papyrus pulling the player back.
    • DefenseEnding(): called at the moment the player leaves their turn. Commonly used to switch turn logic.
    • OnHit(Bullet): called when the player hits a bullet. This handles how a bullet damages the player.

    Overworld main scene (scene_ow_new.lua)¶

    Similar to battle scenes, after significant improvements the main Overworld scene is quite simple. Below are some commonly used functions and features:

    • _CAMERA_:setBounds(minX, minY, maxX, maxY): set camera bounds so the camera does not leave the area.
    • ow.InitMusic("mus_house1.ogg"): set the default Overworld BGM to mus_house1.ogg. If the next scene uses the same BGM, the music will continue without restarting.
    • ow.Init("Maps/ruins_0.lua", "scene_ow_new"): load an Overworld map from a lua tiled file. The second parameter is the scene path.
    • ow.SetBattleScene(): set the battle scene to enter when an encounter occurs.
    • ow.DEBUG: enable Overworld developer mode. This mode is described in the Overworld topic.
    • ow.ENCOUNTER: enable or disable encounters on this map. If disabled, the scene will not encounter monsters.
    • ow.Update(dt): Overworld update logic.
    • ow.getInteractResult("save", 1): called when the player touches an object. Use it to implement a touch + confirm interaction.
    • ow.SaveInteract(): save initialization settings.
    • ow.ChangeScene(): change to another scene or landmark point.
    • ow.ChestInteract(): interact with a chest.
    • ow.dialogNew(): create Overworld-specific dialogs.
    • ow.Draw(): render the Overworld.

    The above content is only the tip of the Overworld iceberg. The Overworld topic will cover it in more detail.

    With the information above, you can start creating your own scene.
