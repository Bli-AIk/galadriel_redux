    Using Maps¶

    Quick note 2.5.1: ow.SetBattleScene now accepts a second argument for the flag.

    After you export a map, the most important step is using it. Let's create an Overworld scene and use the map.

    Initialize the Scene¶

    First, prepare a scene file. I recommend copying the code below exactly as shown.

    Copy and paste: 

    -- This is a template for creating a new scene in the game.
-- You can use this as a starting point for your own scenes.
local SCENE = {}

--_CAMERA_:setBounds(280 - 320, 300 - 240, 280 + 320, 300 + 240)
local ow = require("Scripts.Libraries.Overworld.InitWorld")
ow.InitMusic("mus_gameover.ogg")
ow.Init("Maps/ruins_1.lua", "scene_ow_ruins_1")
ow.DEBUG = true
ow.ENCOUNTER = true
ow.InitEncounter(FLAG.ruins_killed, 80, 40, 3)
ow.SetBattleScene("Battle/scene_battle_ow", "ruins_killed")

-- This is a fake scene for testing purposes.
function SCENE.load()
    -- Load any resources needed for this scene here.
    -- For example, you might load images, sounds, etc.
end

-- This function is called to update the scene.
function SCENE.update(dt)
    -- Update any game logic for this scene here.
    -- For example, you might update animations, handle input, etc.
    ow.Update(dt)

    if (ow.NEXTSTATE ~= nil) then
        ow.CSTATE = ow.NEXTSTATE
        ow.NEXTSTATE = nil
    end
end

-- This function is called to draw the scene.
-- It is called after the main game loop has finished updating.
function SCENE.draw()
    -- Draw the scene here.
    -- For example, you might draw images, text, etc.
    ow.Draw()
end

-- This function is called when the scene is switched away from.
function SCENE.clear()
    -- Clear any resources used by this scene here.
    -- For example, you might unload images, sounds, etc.
    package.loaded["Scripts.Libraries.Overworld.InitWorld"] = nil

    layers.clear()
    ow.Destroy()
end

-- Don't touch this(just one line).
return SCENE

    Let's look at what is happening here.
    First ignore everything after load and focus on initialization: 

    --_CAMERA_:setBounds(280 - 320, 300 - 240, 280 + 320, 300 + 240)
local ow = require("Scripts.Libraries.Overworld.InitWorld")
ow.InitMusic("mus_gameover.ogg")
ow.Init("Maps/ruins_1.lua", "scene_ow_ruins_1")
ow.DEBUG = true
ow.ENCOUNTER = true
ow.InitEncounter(FLAG.ruins_killed, 80, 40, 3)

    • ow = require: loads the Overworld initialization library. Without this, Overworld cannot start.
    • ow.InitMusic("mus_gameover.ogg"): sets the room music to mus_gameover.ogg.
    • ow.Init("Maps/ruins_1.lua", "scene_ow_ruins_1"): loads the Maps/ruins_1.lua map and assigns the scene name scene_ow_ruins_1.
    • ow.DEBUG = true: enables debug mode.
    • ow.ENCOUNTER = true: enables random encounters while the player walks in the map.
    • ow.InitEncounter(FLAG.ruins_killed, 80, 40, 3): sets the encounter system. Enemy spawn count is controlled by FLAG.ruins_killed, starts at 80, has a random range of 40, and spawns 3 encounters.

    It is clear that to use your own map, you just need to change the ow.Init call.

    Player Interaction Objects¶

    Use ow.getInteractResult(type, id) to get the interaction result between the player and an object.

    • type: the object type, such as trigger, chest, save, etc. Use the layer name without the plural form.
    • id: the object's ID, which is set in Tiled to identify the object.

    For example, ow.getInteractResult("trigger", 1) checks whether the player collides with the trigger object with ID 1.

    Example: 

    if (ow.getInteractResult("trigger", 1)) then
    if (keyboard.GetState("confirm") == 1) then
        print("You triggered smth.")
    end
end
    This means if the player touches trigger ID 1 and presses confirm, something happens.

    Enemy Encounters¶

    Set Encounter Info¶

    Use ow.InitEncounter(flag, start, range, amount) to initialize the encounter system.

    • flag: the flag that tracks how many enemies have been defeated (usually a value in FLAG).
    • start: minimum encounter time. The player must walk at least this long before encountering enemies.
    • range: random range. The encounter time is start + random(0, range).
    • amount: the total number of enemies to encounter. The system automatically adjusts the encounter time based on the number of kills. As fewer enemies remain, the time factor increases, but it never exceeds 8x to prevent excessively long play.

    If you are curious about the original formula, here is the description: 

    First define encounter density coefficient ρ:
ρ = min(8, amount / (amount − killed))

As the remaining enemy count (amount - killed) decreases, ρ increases, but it is capped at 8.

Target time threshold:
t = ρ*(base + U(0, range))

U(0, range) is a uniform random number in [0, range].
Recalculate this value after each battle.

    Set the Encounter Scene¶

    Use ow.SetBattleScene(scene, flag) to set the battle scene used for encounters.

    ow.SetBattleScene("Battle/scene_battle_ow", "ruins_killed")
    This means when the player encounters an enemy, they go to Battle/scene_battle_ow, and the encounter count is tracked by FLAG.ruins_killed.

    Special Interactions¶

    Create a Dialogue¶

    Use ow.dialogNew(texts, position) to show a dialogue box with the specified text.

    The position parameter controls whether the box appears above or below. If omitted, the system chooses a suitable position.

    ow.dialogNew({"* Nothing useful here."})

    Create a Chest Interaction¶

    Use ow.ChestInteract(chest) to interact with a chest.

    The chest parameter is the chest key, such as: 

    ow.ChestInteract("chest1")
    This opens CHESTS.chest1 for the player.

    Create a Save Interaction¶

    Use ow.SaveInteract(texts, room, position, direction) to create a save interaction.

    • texts: the text shown when the player saves.
    • room: the room name displayed in the save menu.
    • position: the player's position when the save loads.
    • direction: the player's facing direction when the save loads.

    ow.SaveInteract(
    {
        "[colorHEX:99ffff]* BAKABAKA",
    },
    "Idk where", {121, -455}, "down"
)
    When the player activates the save point, the text BAKABAKA is displayed. The save room name becomes Idk where, the position becomes {121, -455}, and the facing direction becomes down.

    Change Rooms¶

    Use ow.ChangeScene(scene, mark) to switch the player to another room at a specific landmark.

    ow.ChangeScene("Overworld/scene_ow_ruins_1", 1)
