    着色器（Shader）¶

    在Shadertoy网站上看到了好看的shader，但是直接复制到love2d则需要很多改动，而且还不保证最终结果是否一致？

    ShaderToy.lua 库可以完美解决这个问题。

    基本用法¶

    初始化库¶

    local shadertoy = require("shadertoy")

-- 创建单个着色器

local shaderCode = [[
void mainImage(out vec4 fragColor, in vec2 fragCoord)
{
    vec2 uv = fragCoord / iResolution.xy;
    fragColor = vec4(uv, 0.5, 1.0);
}
]]

local myShader = shadertoy.convert(shaderCode)

    使用Shader¶

    假设是直接要用到贴图上的： 

    local shader = myShader.shader
sprite:SetShaders({shader})

    假设要用到绘图函数中（love.draw/SCENE.draw/wave.draw）： 

    function love.draw()
    myShader:apply()
        love.graphics.rectangle("fill", 0, 0, love.graphics.getWidth(), love.graphics.getHeight())
    myShader:clear()
end

function love.update(dt)
    myShader:update(dt)
end

    多通道渲染项目¶

    现在让我们假设有一个shader，它需要有 Buffer A/B/C 的渲染，而这些通常都是独立代码，这时候要如何转化呢？

    答案是，创建一个项目，通过这个项目来存储所有的代码，随后按照逻辑依次渲染绘制。

    创建多通道项目¶

    -- 创建新项目
local project = shadertoy.newProject()

-- 添加渲染通道
project:addPass("bufferA", [[
void mainImage(out vec4 fragColor, in vec2 fragCoord)
{
    // 第一通道着色器代码
    vec2 uv = fragCoord / iResolution.xy;
    fragColor = vec4(sin(uv.x * 10.0 + iTime), cos(uv.y * 10.0), 0.5, 1.0);
}
]])

project:addPass("bufferB", [[
void mainImage(out vec4 fragColor, in vec2 fragCoord)
{
    // 第二通道着色器代码，可以使用前一通道的结果
    vec4 prev = texture(iChannel0, fragCoord / iResolution.xy);
    fragColor = vec4(prev.rg, 0.5, 1.0);
}

]])

project:addPass("final", [[
void mainImage(out vec4 fragColor, in vec2 fragCoord)
{
    // 最终输出通道
    vec4 bufferA = texture(iChannel0, fragCoord / iResolution.xy);
    vec4 bufferB = texture(iChannel1, fragCoord / iResolution.xy);
    fragColor = mix(bufferA, bufferB, 0.5);
}
]])

    更新和渲染项目¶

    在创建项目之后，我们需要把它放到我们的屏幕上：

    function love.update(dt)
    project:update(dt)
end

function love.draw()
    project:draw()
end

    纹理通道设置¶

    在 Shadertoy 中，有些Shader可能会在iChannel0或者其他通道的位置放置一个图片来搭配使用，可能是一个正常的图片，也可能是一个 噪声图 。

    对于 噪声图 ，错误的观点就是这些噪声图总是一个样子的，实际上，每一个噪声图都是随机生成的。

    设置自定义纹理¶

    -- 设置噪声纹理为随机生成的512*512大小的噪声图
myShader:setChannel(0, myShader:generateNoiseTexture(512))

-- 设置图片纹理
local image = love.graphics.newImage("texture.png")
myShader:setChannel(1, image)

    音频纹理支持¶

    -- 设置音频通道（FFT 数据）
myShader:setAudioChannel(2, "music.mp3")

    可用 Uniform 变量¶

    变量名 类型 描述
    iTime number 运行时间（秒）
    iTimeDelta number 帧间隔时间
    iFrame int 帧计数器
    iResolution vec3 分辨率 (width, height, 1)
    iMouse vec4 鼠标位置和点击状态
    iChannel0-3 Image 纹理通道
    iChannelResolution[4] vec3 纹理分辨率

    代码调试¶

    查看转换后的代码¶

    -- 打印最终转换代码
myShader:printCode()

-- 打印原始代码
myShader:printCode("original")

-- 打印 extern 声明
myShader:printCode("extern")

-- 保存代码到文件
myShader:printCode("final", "converted_shader.glsl")

    完整示例¶

    function love.load()
    shadertoy = require("shadertoy")
    -- 创建简单着色器
    local code = [[
    void mainImage(out vec4 fragColor, in vec2 fragCoord)
    {
        vec2 uv = fragCoord / iResolution.xy;
        float r = 0.5 + 0.5 * sin(uv.x * 10.0 + iTime);
        float g = 0.5 + 0.5 * cos(uv.y * 10.0 + iTime);
        float b = 0.5 + 0.5 * sin((uv.x + uv.y) * 5.0 + iTime * 2.0);
        fragColor = vec4(r, g, b, 1.0);
    }
    ]]

    myShader = shadertoy.convert(code)
end

function love.update(dt)
    myShader:update(dt)
end

function love.draw()
    myShader:apply()
        love.graphics.rectangle("fill", 0, 0, 800, 600)
    myShader:clear()
end

    注意事项¶

    1. 语法转换：库会自动将 ShaderToy 语法转换为 LÖVE 兼容格式
    2. 纹理通道：iChannel0-3 对应 LÖVE 的纹理单元
    3. 音频处理：音频通道会实时生成 FFT 和波形数据纹理
    4. 性能优化：多通道渲染使用乒乓缓冲区技术

    这个库使得在 LÖVE 中使用 ShaderToy 着色器变得简单直接，支持大部分 ShaderToy 的特性和语法。

    再次提醒：此库完全不可能保证100%的所有Shader的成功转化。有些Shader是Hack类型，专门为了炸浮点数的精度而制作出来惊人的效果，单独使用此库完全不可能成功复刻。
